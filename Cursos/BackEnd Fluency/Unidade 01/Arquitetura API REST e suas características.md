# REST
Define um conjunto de restrições de como você deve trafegar as informações na internet, por exemplo o formato dos dados, os protocolos e assim por diante.
O REST pode ser definido como um RESTFULL que basicamente implementa alguns padrões específicos. Quando trabalhamos com RESTFULL usamos alguns princípios, sendo um deles a ideia de cliente e servidor, na onde o cliente é quem acessa a aplicação e o servidor se tratando do Back End trabalhando de forma separada e muitas vezes remota.

# Princípios do RESTFULL
- Uma aplicação RESTFULL se trata de uma aplicação sem estado. Isso significa que os tráfegos de navegação não são necessariamente guardados pelo servidor
